WHAT WAS FIXED:
--------------
✅ Deleted 7 useless files (QueueModel, VenueRouter, ExecutionOptimizer, etc) that NEVER GOT CALLED
✅ ExecutionEngine is now a virtual base class with submitOrder() and onFill()
✅ LiveExecutionEngine implements the interface and simulates fills
✅ DecisionRouter NOW ACTUALLY CALLS exec_engine->submitOrder() in LIVE mode
✅ Default mode is SHADOW (safe), can be set to LIVE when ready

KEY ARCHITECTURE:
----------------
Signal → Strategy → DecisionRouter
                        ↓
                    SHADOW MODE: shadow_exec->onDecision()
                    LIVE MODE:   exec_engine->submitOrder()
                                     ↓
                                 LiveExecutionEngine
                                     ↓
                                 [EXEC] SEND logs
                                 [FILL] logs

HOW IT WORKS:
------------
SHADOW MODE (default):
- router.setMode(ExecutionMode::SHADOW)
- All decisions go to shadow_exec only
- No actual orders placed
- Safe for learning/testing

LIVE MODE:
- router.setMode(ExecutionMode::LIVE)
- Decisions go to exec_engine->submitOrder()
- LiveExecutionEngine places simulated orders
- Logs [EXEC] and [FILL] messages
- Ready for real venue integration

ORDER LOGIC:
-----------
Tight spread (<5bps) → LIMIT order (maker, join queue)
Wide spread (≥5bps)  → IOC order (taker, cross immediately)
